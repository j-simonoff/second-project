TERMINAL 

pwd - вывод текущей директории
cd - сменить директорию | cd ~/ - перейти в директорию
ls - вывести содержимое директории | ls -a вывести расширеный список
.. - вернуться на уровень выше 
touch - создать файл: touch %ИМЯ_ФАЙЛА%
mkdir - создать папку | $ mkdir -p dir1/dir-inside/dir-deeper-inside # создали папку dir-deeper-inside в папке dir-inside, которая находится в папке dir1
cp - скопировать файл | cp что_копируем куда_копируем -> cp index.html src/
mv - вырезать
cat - прочитать файл
rm - удалить файл
rmdir - удалить папку
rm -r {имя папки} - удалить папку со всем содержимым
ls -la .ssh/ - вывели список созданных ключей SSH
ssh-keygen - для генерации SSH-пары можно использовать программу 

Указать сразу несколько команд && | mkdir second-project && cd second-project && touch index.html style.css
Дважды нажать Tab - Терминал покажет список всех команд, которые начинаются с этих символов.
Начните печатать имя папки или файла (они должны быть в той же директории) и нажмите Tab. Терминал заполнит имя автоматически.
При навигации в другую директорию. Если ввести cd с названием папки, а затем нажать Tab, в консоль в качестве подсказки выведутся все возможные пути.
Перейти в корневую директорию cd c:/ + Enter или cd /c + Enter.
Перейти в домашнюю директорию cd ~

GIT

git init - загитить папку
rm -rf .git - разгитить папку, удалить скрытую папку .git
git status - проверить состояние репозитория
git add <файлы/папки> - подготовить файлы к сохранению
git add --all - подготовили к сохранению все файлы в репозитории
git add . - добавить всю текущую папку
git commit - выполнить коммит | git commit -m "message" 
git log - просмотреть историю коммитов
git remote add - привязать удалённый репозиторий к локальному
git remote -v - убедиться, что репозитории связаны
git push - отправить изменения на удалённый репозиторий | В первый раз эту команду нужно вызвать с флагом -u и параметрами origin master
git log - вывести лог | git log --oneline - сокращенный лог | строка из цифр и латинских букв после слова commit — это хеш коммита;

Как запушить репозиторий:
cd путь/к/папке       # Перейти в папку проекта
git init               # Инициализировать git в папке
git add .              # Добавить все файлы в индекс
git commit -m "Первый коммит"   # Сделать коммит
git remote add origin https://github.com/username/repo.git   # Привязать удалённый репозиторий
git push -u origin master       # Отправить на GitHub и установить upstream

В числе прочих файлов в папке .git есть служебный файл HEAD. Он указывает на самый свежий коммит.
Вместо хеша последнего коммита можно написать слово HEAD — Git вас поймёт.
